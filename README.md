# Signature Classification: Human vs. Generative AI Models

A PyTorch deep learning project that classifies handwritten signatures as either human-produced or generated by one of three generative AI models (GAN, SDT, VAE). The model processes 2D coordinate sequences from CSV files and performs four-class sequence classification using a Bidirectional GRU network.

---

## Dataset Structure

```
signatures/
├── human/
│   ├── 001g01.csv
│   ├── 001g02.csv
│   └── ...
├── gan/
│   ├── 001g01.csv
│   ├── 001g02.csv
│   └── ...
├── sdt/
│   ├── 001g01.csv
│   ├── 001g02.csv
│   └── ...
└── vae/
    ├── 001g01.csv
    ├── 001g02.csv
    └── ...
```

Each `.csv` file contains space-separated `X` and `Y` columns representing the 2D coordinate sequence of a signature stroke.

---

## Classes & Label Encoding

| Class   | Label | Description                        |
|---------|-------|------------------------------------|
| `human` | 0     | Genuine human handwritten signature |
| `gan`   | 1     | GAN-generated signature             |
| `sdt`   | 2     | SDT-generated signature             |
| `vae`   | 3     | VAE-generated signature             |

---

## Model Architecture

The classifier is a **Bidirectional GRU (BiGRU)** network:

1. **Input**: Variable-length sequence of (X, Y) coordinate pairs, padded/truncated to `MAX_SEQ_LENGTH = 150`.
2. **BiGRU**: 2 stacked layers with 128 hidden units per direction (256 combined). The final forward and backward hidden states are concatenated.
3. **Dropout**: Applied to the concatenated representation (p = 0.25).
4. **Linear classifier**: Maps the 256-dimensional representation to 4 class logits.

---

## Preprocessing Pipeline

For each signature CSV file:
1. Load X and Y columns; drop rows with non-numeric values.
2. Apply **per-signature Min-Max normalization** independently to X and Y axes (range [0, 1]).
3. **Pad** with zeros or **truncate** to a fixed length of 150 time steps.
4. Single-point sequences (uninformative) are zeroed out entirely.

## Usage

### Training

Ensure your dataset is in `./signatures/` (or update `BASE_DATA_DIR` in `train.py`), then run:

```bash
python train.py
```

This will:
- Split the dataset (70% train / 15% validation / 15% test), stratified by class.
- Train the BiGRU model for up to 60 epochs with early stopping.
- Save the best checkpoint (lowest validation loss) to `model.pth`.
- Report final accuracy on the held-out test set.

### Evaluation

```bash
python eval.py
```

When run directly, `eval.py` evaluates using `./signatures/` as data directory and `model.pth` as the checkpoint. To use different paths, modify the variables at the bottom of the script.

For programmatic use, the grading system calls `load_and_predict(directory, model_file)` directly:

```python
from eval import load_and_predict

predictions = load_and_predict("/path/to/signatures", "model.pth")
# Returns: { '/abs/path/to/human/001g01.csv': 0, '/abs/path/to/gan/001g02.csv': 1, ... }
```

## Key Hyperparameters

| Parameter           | Value  | Description                                         |
|---------------------|--------|-----------------------------------------------------|
| `MAX_SEQ_LENGTH`    | 150    | Fixed sequence length (pad / truncate)              |
| `HIDDEN_SIZE`       | 128    | GRU hidden units per direction                      |
| `NUM_RNN_LAYERS`    | 2      | Number of stacked GRU layers                        |
| `DROPOUT_PROB`      | 0.25   | Dropout rate for regularization                     |
| `LEARNING_RATE`     | 0.001  | Initial Adam learning rate                          |
| `WEIGHT_DECAY`      | 1e-4   | L2 regularization strength                          |
| `BATCH_SIZE`        | 32     | Training batch size                                 |
| `NUM_EPOCHS`        | 60     | Maximum training epochs                             |
| `GRADIENT_CLIP`     | 1.0    | Gradient clipping threshold                         |
| `LR_SCHEDULER`      | ReduceOnPlateau | Factor 0.5, patience 7 epochs            |
| `EARLY_STOPPING`    | 15     | Patience (epochs without val loss improvement)      |

**Sequence length tuning results:**

| MAX_SEQ_LENGTH | Val Loss | Val Accuracy |
|----------------|----------|--------------|
| 150            | 0.3348   | 80.21%       |
| 200            | 0.4132   | 78.74%       |
| 400            | 0.3812   | 80.00%       |
| 570            | 0.3520   | 81.89%       |

The value of 150 was selected as the best trade-off between accuracy and training efficiency.

---

## Results

The final model achieves approximately **~80–82% accuracy** on a held-out test set (15% of the full dataset), using `MAX_SEQ_LENGTH = 150`. Performance varies slightly depending on the random seed and dataset composition.
